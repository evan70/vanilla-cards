# Mobile Nav Card

**Mobile navigation panel** for responsive mobile menus. Uses the Dual Toggle Pattern with smooth slide + fade animations.

---

## Overview

The Mobile Nav Card is a **Core component** that provides mobile navigation functionality. It appears when users click the hamburger menu on mobile devices (< 769px breakpoint).

### Key Features

- ✅ **Dual Toggle Pattern** - Hamburger icon animates to X when opened
- ✅ **Smooth animations** - Slide + fade transitions (300ms)
- ✅ **Theme-aware** - Supports light and dark themes via CSS custom properties
- ✅ **Accessible** - ARIA attributes, keyboard navigation (Escape to close)
- ✅ **Overlay backdrop** - Click to close, prevents interaction with content behind

---

## Component Anatomy

```html
<nav class="mobile-nav-card" aria-label="Mobile navigation" hidden>
  <div class="mobile-nav-card__nav">
    <div class="mobile-nav-card__item">
      <a href="#" class="mobile-nav-card__link" data-page="cards">
        <span class="mobile-nav-card__text">Cards</span>
      </a>
    </div>
  </div>
</nav>

<div class="mobile-nav-overlay" hidden></div>
```

| Element | Class | Description |
|---------|-------|-------------|

| Item | `.card--nav-mobile__item` | Individual nav item wrapper |
| Link | `.card--nav-mobile__link` | Clickable navigation link |
| Overlay | `.mobile-nav-overlay` | Backdrop overlay |

## Accessibility

### ARIA Attributes

- `aria-label="Mobile navigation"` on nav element
- `aria-expanded="true|false"` on toggle button

### Keyboard Navigation

| Key | Action |
|-----|--------|
| `Escape` | Close mobile menu |
| `Tab` | Navigate through menu items |
| `Enter` / `Space` | Activate menu items |

### Screen Reader Support

- Menu state announced via `aria-expanded`
- Navigation landmark via `<nav>` element
- Hidden state via `hidden` attribute

## Responsive Behavior

### Breakpoint

```css
@media (min-width: 769px) {
  .card--nav-mobile,
  .mobile-nav-overlay {
    display: none !important;
  }
}
```

**Mobile-first:** Component is hidden on desktop (≥ 769px) and visible on mobile devices.

### Positioning

```css
.card--nav-mobile {
  position: fixed;
  top: var(--header-height);
  right: 0;
  bottom: 0;
  width: 280px;
  max-width: 80vw;
  z-index: 9999;
}
```

## Usage

### Basic Implementation

```html
<!-- Header with mobile toggle -->
<header class="header-card">
  <button class="header-card__mobile-toggle" aria-label="Menu">
    <!-- Hamburger icon -->
  </button>
</header>

<!-- Mobile navigation -->
<nav class="mobile-nav-card" hidden>
  <div class="mobile-nav-card__nav">
    <div class="mobile-nav-card__item">
      <a href="/" class="mobile-nav-card__link">Home</a>
    </div>
  </div>
</nav>

<!-- Overlay -->
<div class="mobile-nav-overlay" hidden></div>
```

### JavaScript Initialization

```javascript
const mobileMenuBtn = document.querySelector('.card--header__mobile-toggle');
const mobileMenu = document.querySelector('.card--nav-mobile');
const mobileOverlay = document.querySelector('.mobile-nav-overlay');

const openMenu = () => {
  mobileMenu.removeAttribute('hidden');
  mobileMenu.classList.add('mobile-nav-card--open');
  mobileMenuBtn.setAttribute('aria-expanded', 'true');
  document.body.style.overflow = 'hidden';
};

const closeMenu = () => {
  mobileMenu.setAttribute('hidden', '');
  mobileMenu.classList.remove('mobile-nav-card--open');
  mobileMenuBtn.setAttribute('aria-expanded', 'false');
  document.body.style.overflow = '';
};
```
